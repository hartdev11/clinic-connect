# Phase 5: กฎข้อมูล AI & LINE (Spec)

เอกสารนี้กำหนดชัดเจนว่า **ข้อมูลประเภทไหนส่งให้ใครได้** — ใช้เป็นกฎสำหรับ AI Agents และการส่ง LINE

---

## 5.1 ข้อมูลที่ส่งให้ลูกค้าได้

| ประเภท | ตัวอย่าง | หมายเหตุ |
|--------|----------|----------|
| **แชท** | การพูดคุยกับลูกค้า, ตอบคำถาม, แชท LINE/FB/IG | ได้ |
| **โปรโมชัน** | เสนอโปรโมชันที่เหมาะกับลูกค้า | ได้ |
| **นัดก่อนทำ (Pre-care)** | เตือน/ปรึกษาก่อนเข้ารับบริการ | ได้ — **ไม่เจาะลึกการแพทย์** |
| **จองคิว** | ยืนยันนัด, วันที่ว่าง, บริการที่จอง | ได้ |

**ข้อห้าม:** ไม่พูดเจาะลึกเรื่องการแพทย์กับลูกค้า (วินิจฉัย, แนะนำการรักษาเชิงลึก ฯลฯ)

---

## 5.2 ข้อมูลที่ส่งให้คลินิก (LINE เท่านั้น) — ไม่ส่งหาลูกค้า

| ประเภท | ตัวอย่าง |
|--------|----------|
| **วิเคราะห์ข้อมูล** | สถิติแชท, ยอดจอง, รายได้, แนวโน้ม |
| **แจ้งเตือนภายใน** | แชทรอตอบ, นัดที่ใกล้ถึง, เตือนที่ต้องตรวจ |
| **สรุปรายวัน/รายสัปดาห์** | รายงานส่งไป LINE คลินิก |

**ห้ามส่งข้อมูลวิเคราะห์/สถิติ/รายงานภายในไปหาลูกค้าเด็ดขาด**

---

## 5.3 (ภายหลัง) Webhook/API ส่งไป LINE

- สร้าง Webhook หรือ API ที่ส่งข้อความไป LINE ของคลินิก
- ใช้สำหรับ: Analytics Agent, LINE Notify Agent ส่งสรุป/แจ้งเตือน

---

## อ้างอิงในโค้ด

- ข้อมูลส่งลูกค้า: แชท, โปรโมชัน, นัดก่อนทำ, จองคิว — ดู `docs/AI-DATA-SPEC.md` § 5.1  
- ข้อมูลส่งคลินิก (LINE): วิเคราะห์, สถิติ, แจ้งเตือน — ดู `docs/AI-DATA-SPEC.md` § 5.2
